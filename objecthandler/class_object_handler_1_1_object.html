Detailed Description

Interface for Objects to be stored in the ObjectHandler Repository.

Objects are constructed by the client and passed to Repository::storeObject() for storage in the Repository.

All member functions of this class are implemented inline. This is not for performance reasons; the entire implementation of the class must appear in the header file in order to allow derived classes to inherit from Object across DLL boundaries on the Windows platform.

◆ permanent()

const virtual bool& ObjectHandler::Object::permanent ( ) const
inlinevirtual

Query the value of the "permanent" flag.

Permanent Objects remain in the Repository after a call to Repository::deleteAllObjects() or Repository::collectGarbage().

This feature allows a finer level of granularity in maintaining the contents of the Repository.
